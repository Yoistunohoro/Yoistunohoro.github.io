C语言类型小结

C语言笔记

常用的format字符串中的格式符

%d   对应参数是一个 int 类型
%f   对应参数是一个 float 类型
%ld   对应参数是一个 long int 类型
%lld   对应参数是一个 long long int 类型
%s   对应参数是一个 const char* 类型
%u   对应参数是一个 unsigned int 类型
%zu   对应参数是一个 size_t 类型
%td   对应的参数是一个 ptrdiff_t 类型
%x   对应的参数是一个 int 类型，不过会以16进制输出

int类型

  int 类型在64位和32位系统中均为4字节即32位，且默认为signed类型。
最大值为0x7FFFFFFF，最小值为0x80000000。

int类型对应的整数字面量可直接按照自然方式书写

int i = 0、 -128、 127、 +2333; //都为正常赋值

八进制整数字面量以0打头

int i = 01、023、-0123; //都为8进制赋值

十六进制整数字面量以0x打头

int i = 0x123、-0x0045、0xabcdef; //都为16进制赋值

无符号类型字面量书写可在字面量后面添加U或者u(一般书中为大写)

unsigned int i = -128U、127U、01U;
unsigned i = 1; //声明一个无符号类型字面量时int可以省略不写

short类型

  signed short int类型，简写为short类型，常称为短整型，在32位和64位的系统中均占用2个字节即16位。
最大值为0x7FFF，最小值为0x8000。
unsigned short int 类型最大值为0xFFFF，最小值为0x0000。
unsigned short和short类型没有特别的整数字面量，可直接使用int和unsigned int类型的整数字面量赋值即可。

long类型

  signed long int类型，简写为long类型，常称为长整形，在32位系统下占用4个字节即32位，但在64位系统下，根据编译器的不同，占用4个字节32位或者8个字节64位，可用sizeof(long)查看其大小。

定义时可以在字面量后添加L和UL

long i = 100;
long i = 1000L;
long i = 0x10000000L; //由于0x10000000已经超过int类型字面量所能表示的最大值，所以在字面量后加上L表明该字面量是长整形
//某些编译器如果不在后面加上L，可能会出现报错，或在运算时会丢失数据
unsigned long i = 100000UL; //表示一个无符号长整形
在写程序时，由于不同的环境下long类型的类型宽度不同，我们在定义一个整数类型时尽量不要使用long类型。

long long类型

  signed long long类型，简写为long long类型，在32位和64位操作系统下，均占用8个字节即64位。

定义时可以在字面量后面添加LL和ULL

long long i = 100;
long long i = 10000LL;
long long i = 0x100000000LL; //同理long类型
unsigned long long i = 100000ULL;

bool类型

  _Bool类型对象只占用一个字节。

bool a = true;
bool b = false;
bool c = a == b; //使用a == b的结果给C赋值，这里C为0
c = 10 > 5; //这里C为1
a = 0x10000000; //这里a为1
b = 0.0; //这里b为0
c = -1; //这里c为1
在最后的两行代码中，对于a = 0x10000000，不是简单的对字面量进行高位截取，只取最低位的结果，而是将0x10000000 != 0的结果赋值给了a，对于b = 0.0也同样是将0.0 != 0.0的值赋值给了b，就是我们常说的非零为真。
